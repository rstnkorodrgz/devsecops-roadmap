# 🏔️ Post-Certification Technical Ladder — The Four Pillars

> **Phase 6, effectively — but open-ended.** Past senior level, certifications stop differentiating; **public artifacts do**. The pillars turn the quarterly artifact rule from a KPI into a system.
>
> **When to start:** ramp Pillars A and D lightly from Phase 3 onward (they feed the phase artifacts); go full-cadence after CCSP.

## The pillars

| Pillar | Focus | Cadence / target |
|---|---|---|
| [A — Open Source](pillar-a-open-source.md) | CNCF security ecosystem contribution | ONE project, deep: docs → merged PRs → recognized contributor |
| [B — Research & Speaking](pillar-b-research-speaking.md) | Public technical voice | 2 talks/year + written version of each |
| [C — AI Security](pillar-c-ai-security.md) | The specialization bet | Portfolio-driven: harness + secure RAG + CI/CD gates |
| [D — Architecture Depth](pillar-d-architecture.md) | Staff-level design artifacts | Reference architectures + quarterly sanitized design doc |

## 📏 The quarterly artifact rule (roadmap-wide KPI)

Every quarter ships **at least one** public artifact: a merged PR, a talk, a written deep-dive, or a reference architecture. During Phases 1–5 the phase artifact satisfies this; after that, the pillars supply it.

### Quarterly KPI checklist (copy into [progress/quarterly-tracker.md](../progress/quarterly-tracker.md))
- [ ] 1 public artifact shipped this quarter (PR / talk / post / architecture)
- [ ] Next exam has a **booked date**
- [ ] GitHub contribution graph shows sustained activity (no cert-cramming gaps)
- [ ] LinkedIn updated with each certification + artifact
